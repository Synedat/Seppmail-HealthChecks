# Homepage structure

This repository is prepared so it can be presented both as a GitHub repository and as a simple GitHub Pages landing page.

## Recommended sections
1. Hero message with Synedat-as-delivery-contact positioning and a one-line value statement
2. Product image block with explicit image origin note
3. Quick links to quick start, architecture, permissions, operations and workshop kit
4. Customer use cases such as health checks, pilot preparation, post-change validation and evidence capture
5. Trust block with official SEPPmail references and source notes
6. Synedat section with contact path and related repositories

## Hero message example
**Seppmail-HealthChecks** helps customer teams turn SEPPmail-related operational checks into structured workshops, pilot baselines and delivery-ready health evidence with **Synedat** as the engineering and implementation contact.

## Search-friendly phrases
SEPPmail health checks, secure email operations, SMTP TLS validation, operational readiness, Synedat
