# Workshop kit

## Workshop outcome
Help participants understand how to turn SEPPmail-related operational checks into a structured readiness, remediation and implementation discussion.

## Target audience
Operations teams, service owners, architects, platform engineers and customer stakeholders involved in secure email operations.

## Suggested agenda
1. Customer context and operational problem statement
2. Product context and SEPPmail-related touchpoints
3. Health-check scope and least-privilege approach
4. Example run or demo output
5. Findings, readiness scoring and evidence handling
6. Prioritized next steps for pilot, hardening or handover

## Materials
- README and landing page
- `docs/ARCHITECTURE.md`
- `docs/RBAC-AND-PERMISSIONS.md`
- `docs/SECURITY-AND-COMPLIANCE.md`
- `docs/EXAMPLE-OUTPUTS.md`
- `examples/demo-walkthrough.md`
- `docs/SEPPMAIL-REFERENCES.md`

## Good workshop questions
- Which checks can be run read-only first?
- Which findings block a pilot or production handover?
- Which evidence must be retained for change or audit reviews?
- Which customer-specific extensions should Synedat add next?
